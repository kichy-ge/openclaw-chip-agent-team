# 贡献指南

[English](CONTRIBUTING.md) | [简体中文](CONTRIBUTING.zh-CN.md)

欢迎贡献。

## 欢迎的贡献方向

比较适合这个仓库的贡献包括：

- 改进角色契约
- 改进角色 playbook
- 提升配置模板的安全性和可复用性
- 增加高质量样例讨论
- 改进安装脚本或初始化脚本
- 补充如何适配不同芯片项目的文档

## 不要提交的内容

请不要提交这些内容：

- 真实 API key
- 真实客户或项目聊天记录
- 从生产 / 私有 OpenClaw 工作区里复制出来的 memory
- 暴露保密信息的截图、音频、附件或日志

## 风格要求

- 文档尽量简洁、实用
- 角色边界要清楚
- 不要做假的能力宣传
- 尽量显式写出假设和 owner
- 模板应尽量对多个芯片项目都可复用

## 推荐的 PR 形态

小而清晰的 PR 最容易 review：

1. 一个角色定义改进
2. 一个安装 / 配置改进
3. 一个文档 / 示例改进

## Review 标准

一个改动是否值得合并，通常看这几件事：

- 它有没有让角色更具体？
- 它有没有提升真实协作质量？
- 它是否仍然安全可公开？
- 它是否仍然具有跨项目复用性？
