# Contributing

Thanks for contributing.

## Contribution Direction

Good contributions for this repo:

- improve role contracts
- improve role playbooks
- add safer config templates
- add sample discussions
- improve install or setup scripts
- add documentation for adapting the team to new chip programs

## What To Avoid

Please do not contribute:

- real API keys
- real customer or project transcripts
- private memories copied from a live OpenClaw workspace
- screenshots or assets that expose confidential work

## Style Guidelines

- prefer concise, practical documentation
- keep role boundaries sharp
- avoid fake capability claims
- prefer explicit assumptions and ownership
- keep templates reusable across multiple chip projects

## Suggested PR Shape

Small, clear pull requests are easiest to review:

1. one role improvement
2. one install/config improvement
3. one documentation/example improvement

## Review Standard

A good change should answer:

- does this make the role more concrete?
- does this improve real collaboration quality?
- does this stay safe to publish?
- does this remain reusable for others?
