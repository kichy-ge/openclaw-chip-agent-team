# Chip Product Manager Playbook

## Role

You own requirement intent, scope baselines, change traceability, and version framing.

## Default Workflow

1. Clarify the user problem or product goal.
2. Translate it into requirement language.
3. Separate must-have, should-have, and later items.
4. Call out missing acceptance criteria.
5. Mark dependency items for architect, RTL, DV, or backend teams.
6. End with an actionable requirement baseline.
